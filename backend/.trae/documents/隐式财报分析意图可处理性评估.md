# 隐式财报分析意图可处理性评估

## Summary

评估问题：  
新增专用 skill 后，用户用隐式表达（例如“分析贵州茅台半年报中的投资价值”“分析贵州茅台半年报的资产负债表”）是否能被系统正确处理？

结论：

1. **当前实现下，不能稳定处理**这类隐式请求。  
2. 原因是现有自动路由会优先命中通用 `pdf` skill（表单结构提取），不会自动串联“财报数据提取 -> 财报分析”链路。  
3. 通过新增“意图分流 + 多 skill 编排”后，可稳定支持该类请求。

## Current State Analysis

### 1) 当前隐式请求的命中路径

- 用户消息含“贵州茅台半年报”会被 `document-router` 文件名匹配命中 PDF。
- 命中后自动路由到现有 `pdf` skill（`extract_form_structure.py`）。
- 该输出并非财报数值，不足以支撑“投资价值/资产负债分析”。

=> 现状会“看起来触发了 skill”，但业务目标未满足。

### 2) 两个示例请求的可处理性

#### 示例 A：分析贵州茅台半年报中的投资价值

- 语义范围通常覆盖：盈利能力、现金流、估值、成长、风险，不仅是资产负债表。
- 若系统只有“资产负债表分析” skill，最多给出资产质量与偿债维度结论，不能完整覆盖“投资价值”。

评估：**部分可处理，语义存在超范围风险**。

#### 示例 B：分析贵州茅台半年报的资产负债表

- 与目标 skill 高度匹配，只要能抽取结构化财务数据即可处理。

评估：**可处理（前提是有稳定的数据提取与字段标准化）**。

## Proposed Changes

说明：本阶段仅评估方案，不执行代码。

### A. 增加“隐式意图识别层”（必须）

在 chat 编排中增加财报意图标签：

- `intent=financial_data_extraction`
- `intent=balance_sheet_analysis`
- `intent=investment_value_analysis`

识别信号：

- 文件名命中（如“贵州茅台半年报”）
- 任务词命中（“分析/投资价值/资产负债表/偿债能力/资产质量”）

### B. 引入“多 skill 串联编排”（必须）

推荐流程：

1. 命中文件 + 分析意图
2. 先调用 `pdf_financial_extract`
3. 再调用 `financial_statement_analysis`
4. 汇总结果给 LLM 生成最终回答

关键点：

- 不是二选一触发，而是“有条件串联调用”。
- 失败时要返回结构化阶段错误（extract/analyze）。

### C. 投资价值请求的范围控制（必须）

对于“投资价值”语义：

- 若当前仅有资产负债表分析能力，应明确提示“当前结论基于资产负债表维度，不含利润表/现金流/估值模型”。
- 可选扩展路线：
  - `income_statement_analysis`
  - `cashflow_analysis`
  - `valuation_calc`

### D. 路由策略调整建议

避免“文件名命中即固定走通用 pdf skill”：

- 当命中“财报分析意图”时，优先走财报专用提取 skill。
- 仅在“纯读取 PDF 结构/表单处理”时走原 `pdf` skill。

## Assumptions & Decisions

- 决策 1：隐式请求需要“意图识别 + 串联执行”，单次路由不足以覆盖。
- 决策 2：v1 对“投资价值”采用范围受限回答（先聚焦资产负债表）。
- 假设 1：用户可接受“能力边界声明”。
- 假设 2：项目优先保证可解释和稳定，不追求一步到位全财报估值。

## 风险评估

1. **高风险：意图识别误判**
   - 可能把普通 PDF 阅读误判为财报分析流程。
   - 对策：关键词 + 文件上下文 + 置信度阈值。
2. **高风险：投资价值语义超出当前能力**
   - 回答可能“看似完整、实际缺关键维度”。
   - 对策：强制输出覆盖范围声明。
3. **中风险：多 skill 编排失败链复杂**
   - 任一步失败可能中断。
   - 对策：统一阶段错误码与降级策略。

## Verification Steps

执行阶段建议验收：

1. 输入“分析贵州茅台半年报的资产负债表”：
   - 自动串联 `pdf_financial_extract -> financial_statement_analysis`。
2. 输入“分析贵州茅台半年报中的投资价值”：
   - 返回分析结果并附能力范围声明。
3. 输入“读取贵州茅台半年报结构”：
   - 仍可走通用 `pdf` skill（不被误改）。
4. 任一步失败时返回结构化错误（含 `failureStage`）。
